# --- CUDA-enabled PyTorch ----------------------------------------------------
# On Linux this pulls the CUDA wheel from the PyTorch index automatically.
# On macOS the URL has no Darwin wheels, so pip silently falls back to PyPI
# and installs the regular CPU/MPS wheel — no action needed there.
#
# Pick the URL that matches BOTH your driver's CUDA version AND your GPU's
# compute capability. Full matrix: https://pytorch.org/get-started/locally/
#   cu118  →  CUDA 11.8 driver, sm_50 .. sm_90 (Maxwell .. Hopper)
#   cu121  →  CUDA 12.1 driver, sm_50 .. sm_90
#   cu124  →  CUDA 12.4 driver, sm_50 .. sm_90
#   cu126  →  CUDA 12.6 driver, sm_50 .. sm_90
#   cu128  →  CUDA 12.8 driver, sm_50 .. sm_120  (Blackwell: RTX 50-series, B100, etc.)
#   cu130  →  CUDA 13.0 driver, sm_75 .. sm_120
#
# Check your server with:
#   `nvidia-smi`                                   → driver CUDA version
#   `python -c "import torch; print(torch.cuda.get_device_capability(0))"`
#                                                  → (major, minor) compute capability
# RTX 5090 / 5080 etc. are sm_120 → need cu128 or cu130. Older GPUs work with
# any cu* wheel.
#
# NOTE: must be `--index-url`, not `--extra-index-url`. Otherwise pip can
# silently fall back to the cu130 wheel on PyPI and you get a CUDA driver
# mismatch at `torch.cuda.is_available()`.
--index-url https://download.pytorch.org/whl/cu128
--extra-index-url https://pypi.org/simple
